# random-pointer-list-serialization
Тестовое задание: сериализация и десериализация двусвязного списка с произвольными указателями на C++.

Комментарии к техническому заданию:

"ListNode модифицировать нельзя"
:
Основа структуры это сырые указатели, поэтому я не могу обойтись хотя бы без деструктора. Но, по хорошему, здесь не обойтись без Правила пяти.

"Реализуйте сериализацию и десериализацию двусвязного списка ListNode"
:
ListNode как по структуре, так и по наименованию не двусвязный список, а только его узел. Двусвязный список это контейнер на один уровень абстракции выше. Из этого не совсем понятно необходимо/можно ли было писать оболочку для удобства. Поскольку ранее было сказано, что модифицировать его нельзя, то включение структуры в класс более высокого уровня, мне показалось, подходит под модифицирование, поэтому этого делать не стал, пусть это было бы более органично и правильно.
